Swift 中的 Any Anyobject optional

Any 和 Anyobject

因为编译的时候不会对 id 类型的变量进行类型检查，可以表示任意实例变量。在 swift 中相对应的是 Anyobject，为了对 cocoa 框架中很多方法做出兼容。
因为 id 可以指向 nil，所以一个 API 的返回值为 id 的时候，对应的 swift 里面返回的是 Anyobject?。

Any 可以指向 class，struct 和 enum 等任意类型，Anyobject 只能指向 class 类型

optional

swift 中 nil 也是一种类型，所以一个 string 类型的变量不能赋值为 nil。
但是对于一个 string? 类型（跟 string 完全是两个类型），既可以接收一个字符串，也可以赋值为 nil。

  1. 使用 ！对一个可选类型强制解包是不安全的，需要保证在取值的时候不会为 nil，如果为 nil 则程序直接崩溃。想强制解包的话，需要在之前判断是否为 nil。

  2. if let 解包的就类似加了判空的强制解包，能保证安全。

  3. 可选调用链的话，在其中一个值为 nil 时，整个都为 nil，例如 person?.name?.uppercaseString

  4. 空合运算符，let newName = name ?? “no name”，意思就是 name 有值的话，就赋值为 name!，否则为 “no name”。

  5. 隐式可选型，var name: String! = “name”，作用就是在这个类没有被初始化的时候，他是没有值的，当初始化之后，可以确保他是有值的，在使用的时候同上面的可选型一样，需要判空。

一般解包的时候用 if let 或者 guard let 就好了
