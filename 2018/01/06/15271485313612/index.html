git学习笔记

  1. 集中式版本控制：CVS，SVN，分布式：git
  2. git的优势是在强大的分支管理
  3. git add 添加工作区的文件到暂存区
  4. git commit 把暂存区的所有更改提交更改到仓库
  5. git status 是用来查看当前仓库的状态
  6. git diff 是用来查看工作区和暂存区的比较，git diff --cached是比较暂存区和分支的比较
  7. git log 是用来查看提交日志的，可以添加--pretty=oneline可以简化日志显示
  8. git reset 是用来回退版本，当前版本为HEAD，上个版本为HEAD^，上上个版本为HEAD^^，往上一百个版本为HEAD~100
  9. git reflog 是用来查看每一次更改的历史命令
  10. 在创建git版本库的时候，自动创建了一个master分支，工作区（Working Directory）是我们电脑上项目的文件夹，在工作区下有个.git的隐藏目录，里面有暂存区（stage）和一个分支master，以及指向master的一个指针HEAD
  11. git check -- <file>的意思就是撤销工作区修改，如果修改后被放到暂存区，则撤回到暂存区版本，如果修改后没有被放到暂存区，则退回到版本库版本
  12. 撤销暂存区修改可以先用git reset HEAD <file>把暂存区的修改放回工作区，再用git check -- <file>命令把工作区的修改撤销
  13. 删除文件git rm <file>，误删的文件恢复git check -- <file>
  14. 在关联远程仓库之后，本地commit之后，如果有必要的话，用git push origin master推送到远程服务器上面，origin是远程仓库的名字，master是分支名称
  15. git clone是把远程仓库克隆到本地，Git支持多种协议https，ssh
  16. git checkout -b dev就是创建dev分支，然后切换到dev分支，相当一git branch dev和git checkout dev两条命令，git branch查看当前分支情况
  17. git merge dev是把名字叫dev的分支合并到当前分支上
  18. git branch -d dev是把叫dev的分支给删除
  19. 合并分支的时候可能会有冲突，这样的情况下就是先解决冲突，再重新commit
  20. 合并分支的时候，有时候会触发fast foward合并，只是简单的把指针改变一下，并没有生成commit信息，可以使用git merge --no-ff强制不使用fast foward
  21. 可以使用git stash把手头的工作先保存一下，去干其他事情，然后可以再用git stash pop回来。可以使用git stash list查看被保存的工作
  22. 查看远程库信息git remote -v
  23. 多人远程推送，本地推送git push origin <branch-name>，如果推送失败，要使用git pull拉取最新版本，如果有冲突，先解决冲突，并在本地提交，并再次尝试git push origin <branch-name>
  24. 在本地创建和远程分支对应的分支git checkout -b branch-name origin/branch-name
  25. 建立本地分支和远程分支关联，使用git branch --set-upstream branch-name origin/branch-name
  26. 使用命令git tag <tagname>用于新建一个标签，默认为HEAD，使用git tag -a <tagname> -m "xxxxx"可以指定标签信息
  27. 标签一般都是在本地，使用命令git push origin <tagname>可以推送一个本地标签，git tag -d <tagname>是删除一个本地的标签，git tag origin :refs/tags/<tagname>可以删除一个远程标签
  28. 使用github，可以先fork一个项目，在自己账号下生成一个同样的仓库，然后clone这个仓库到本地，就可以提修改或者新增功能，然后再推送到自己的仓库，可以向原来的仓库发起pull request，别人可以选择是否接受
