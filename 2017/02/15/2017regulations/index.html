hexo部署和更新博客

  1. 有时候会出现hexo server不能使用的情况，这是因为hexo3中server被分离出来，需要先sudo npm install hexo-server,或者检查一下server模块是不是安装在博客目录下，当前是不是在博客目录下

  2. 安装自动部署工具$ npm install hexo-deployer-git --save

  3. 更改hexo服务器端口号$ hexo server -p 5000，从默认的4000更改到5000

  4. 修改主题方法是两步，先下载主题然后放到博客中的themes文件夹（专门用来存放主题）下，再修改主题的配置文件_config.yml中相关参数，启用themes文件夹下新增的主题,例如

    1. 下载Next主题:

      进入 hexoBlog/themes 文件夹中，打开Git Bash面板，输入：
      git clone https://github.com/iissnan/hexo-theme-next themes/next
      把主题包克隆到themes文件夹中即可。

    2. 启用主题:

      与所有 Hexo 主题启用的模式一样。 当 克隆/下载 完成后，打开themes下的主题配置文件_config.yml， 找到 theme 字段，并将其值更改为 next（注意冒号后面要留一个 空格）。

    3. 验证主题:

      清除并重新生成hexo静态文件，启动本地服务器，然后通过http://localhost:4000 预览博客：

      hexo clean         #清除静态文件
hexo g          #重新生成静态文件
hexo s         #启动服务器

  5. 每次更新博客

    • 书写博客md文件
    • 进入XXX.github.io文件夹，hexo server或者hexo s运行服务器，预览
    • 审核无误后运行hexo clean && hexo g && hexo d(清除文件，生成新文件，部署到github服务器)

  6. 更改地址到自定义域名，登录github，进入到博客仓库，点击setting，选择custom domain ，输入要自定义的域名，更改保存，每次hexo d之后都要进行这个操作
